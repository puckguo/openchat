# OpenCode 多人协作 AI 聊天室

> 让团队协作更智能，让 AI 成为你的第七个队友

---

## 为什么选择 Open CoChat？

在 Discord、飞书、钉钉之外，你是否需要一款：
- **专为开发者设计**的协作空间
- **内置 AI 助手**，能真正读懂代码、执行命令
- **完全开源**，数据自主可控
- **部署简单**，一台云服务器即可

---

## 核心功能

### 1. 智能对话，不止聊天

```
你: @ai 帮我看看 src/api.ts 里的错误

AI: [正在思考...]
    [正在使用工具...]

    我已读取 src/api.ts，发现第42行存在类型错误：
    getUserById 返回了 User | null，但调用处未处理空值情况...

    ✓ 已为你生成了修复代码，是否应用？
```

**AI 能力清单：**
- 读取、编辑、搜索项目文件
- 执行终端命令（支持权限控制）
- 自动总结长对话，保持上下文连贯
- @ai 触发，像@同事一样自然

### 2. 团队协作，权限清晰

| 角色 | 权限 |
|------|------|
| Owner | 所有权限，可设置会话密码 |
| Admin | 管理成员、删除消息 |
| Member | 发送消息、使用 AI |
| Guest | 只读访问 |

### 3. 数据安全，自主可控

- 阿里云 OSS 存储文件，随时访问
- PostgreSQL 持久化聊天记录
- 支持会话密码保护
- 开源代码，私有部署

### 4. 一键部署，全球可达

```bash
# 3分钟启动你的协作服务器
bun install
bun run start

# 朋友连接地址
ws://your-server:3002
```

## 使用场景

### 场景一：远程团队协作

> 5人团队，分散在不同城市，需要一个即时沟通、AI 辅助编码的空间

**解决方案：**
- 云服务器部署 Open CoChat
- 每人一个角色权限
- @ai 帮忙审查代码、生成文档

### 场景二：技术学习小组

> 一群学习编程的朋友，需要互相讨论、问 AI 问题

**解决方案：**
- 本地或云服务器快速部署
- 语音消息转文字，方便回顾
- AI 自动总结学习要点

### 场景三：个人知识管理

> 开发者需要一个 AI 驱动的笔记+代码助手

**解决方案：**
- 私有部署，数据安全
- AI 记住你的代码风格
- 文件即写即存，永不丢失

## 快速开始

### 1. 安装依赖
```bash
bun install
```

### 2. 配置环境
```env
# 数据库（可选，用于持久化）
VITE_RDS_HOST=your-rds-host
VITE_RDS_DATABASE=opencode-chat

# 文件存储（可选）
VITE_OSS_ACCESS_KEY_ID=your-key
VITE_OSS_BUCKET=your-bucket

# AI 助手
DEEPSEEK_API_KEY=sk-your-key
```

### 3. 启动服务器
```bash
bun run start
```

### 4. 邀请朋友
```
地址: ws://your-server:3002
会话: team-dev
密码: [可选]
```
